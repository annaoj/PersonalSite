Git Cheat Sheet

Anna Jimenez
on Jan 6, 2019

Git is one of the most popular distributed version control system that supports collaboration within developers.I decided to create this cheat sheet to provide a quick reference to commands that are useful for working and collaborating in a Git repository.

Ther are so many commands and when should you use it? I think the best way to divide it by actions

I will constantly update this post based on my experience with Git.

1.Initialize a repo

Create an empty git repo or reinitialize an existing one


  • $ git init

2.Check status

Not sure what you've done?Git status will show you.


  • $ git status

3.Clone remote repository

  • $ git clone https://github.com/annaoj/SDPre.git

4. Commit your changes

  • $git add filename.java   or if you want to add all changes   $git add --all

  • git commit -m "message"

  • git push origin master

5.Get changes from remote repo

You nee to use pull and make sure name of the branch is correct.


  • git pull origin branchname
